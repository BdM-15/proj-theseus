# Win Theme Identification User Query Prompt

**Purpose**: Extract competitive differentiators and transform them into compelling win themes that link customer benefits to your discriminating features with proof.

**Target User**: Capture managers developing win strategy, proposal writers incorporating themes into narrative, color team reviewers validating theme effectiveness.

**Value Proposition**: Shipley-based win theme methodology transforms generic capability statements into persuasive, evidence-based differentiation. Win themes are the bridge between "what we do" and "why you should care" - the difference between compliance and competitiveness.

**Shipley Methodology**: Win theme development per Shipley Proposal Guide - "Theme statements link a customer benefit to the discriminating features of your offer. Strategies = things to do. Themes = things to say."

---

**CAUTION**: Use sparingly. Ghost themes subtly highlight competitor weaknesses without naming them.

```

[Your capability - e.g., "ISO 9001:2015 certified with documented QC procedures"]

```

**Shipley Ghost Theme Principles**:

- вң… DO: Emphasize your strength that competitor lacks
- вң… DO: Frame as customer benefit ("You will experience X because we have Y")
- вқҢ DON'T: Name competitors or directly attack them
- вқҢ DON'T: Use ghost themes in executive summary (too obvious)
- вқҢ DON'T: Ghost a weakness without solid competitive intelligence (don't guess)

**Action Caption Best Practices**:

- Start with customer benefit (not "Our process is...")
- Use active voice and strong verbs (Reduces, Ensures, Delivers, Minimizes, Achieves)
- Include proof/metrics when possible (60%, 99.7%, 15 years)
- Keep to 1-2 lines max (captions, not paragraphs)

### 8. THEME VALIDATION CHECKLIST

**Purpose**: Ensure themes meet Shipley quality standards before incorporating into proposal.

```

вң… CUSTOMER BENEFIT (answers "So what?"):

   в””в”Җ [Is benefit specific/measurable, not generic fluff?]

   в””в”Җ [Is this a REAL differentiator competitors can't easily match?]

   в””в”Җ [Do you have concrete proof - metrics, testimonials, certifications?]

   в””в”Җ [Can reader understand theme in 10 seconds or less?]

   в””в”Җ [Does theme appear in exec summary + relevant sections?]
```

## Metadata Fields to Query

Use these exact field names when identifying win themes:

| Field Name                | Entity Type        | Description                                          |
| ------------------------- | ------------------ | ---------------------------------------------------- |
| `agency_pain_points`      | RFP_METADATA       | Customer hot buttons from Section C/M analysis       |

**Relationship Types to Traverse**:

- `REQUIREMENT` -[:EMPHASIZES]-> `CUSTOMER_PAIN_POINT`

- `PAST_PERFORMANCE` -[:PROVES]-> `COMPANY_CAPABILITY` (evidence)
- `COMPETITOR` -[:LACKS]-> `COMPANY_CAPABILITY` (ghost theme opportunities)

- Discriminator matrix comparing capabilities

**User Query**: "How do I incorporate win themes into [Section X]?"

## Important Reminders

### What This Prompt DOES:

вң… Generate customer-focused win themes (benefit + discriminator + proof)  
вң… Map themes to proposal sections for writer guidance  
вң… Provide action caption examples for visual reinforcement  
вң… Identify competitive differentiators through matrix analysis  
вң… Validate theme quality using Shipley standards  
вң… Create ghost themes for subtle competitive attacks  
вң… Transform generic capabilities into persuasive differentiation

### What This Prompt DOES NOT DO:

вқҢ Write full proposal content (themes are framework, not complete narrative)  
вқҢ Replace competitive analysis (themes require understanding of competitors)  
вқҢ Guarantee proposal will win (themes enhance competitiveness, don't ensure it)  
вқҢ Create differentiation where none exists (can't invent discriminators)

**Shipley Philosophy**: "Strategies = things to do. Themes = things to say. Themes link customer benefits to your discriminators with proof. Without proof, it's just marketing fluff."

---

## Quality Indicators

**High-Quality Response Includes**:

- Specific customer benefits (not "quality service" but "99.7% on-time delivery")
- True discriminators (capabilities competitors lack or can't prove)
- Concrete proof (metrics, CPARs, certifications, customer testimonials)
- Theme-to-section mapping showing WHERE to use each theme
- Action captions that reinforce themes visually
- Competitive differentiation matrix showing why themes work

**Low-Quality Response Includes**:

- Generic themes anyone could use ("committed to excellence")
- No proof points (claims without evidence)
- Company-centric language ("we are the best" vs. "you will receive")
- Themes not linked to evaluation criteria (don't align with scoring)
- No competitive differentiation (doesn't explain why you beat competitors)

## Shipley Proposal Guide Integration

**Key Principles** (Shipley Proposal Guide, p. 125-140):

1. **Theme Formula**: WIN THEME = CUSTOMER BENEFIT + DISCRIMINATOR + PROOF
2. **Strategies vs. Themes**: Strategies = what to do. Themes = what to say.
3. **Integration**: Themes must appear in exec summary, volume openers, section openings, action captions
4. **Proof Required**: Themes without evidence are just marketing claims
5. **Competitive Focus**: Themes must differentiate (not just comply)

**Theme Development Process** (Shipley):

1. Analyze customer (hot buttons, pain points, evaluation criteria)
2. Inventory discriminators (what sets you apart - capabilities, experience, metrics)
3. Gather proof (past performance, CPARs, certifications, testimonials)
4. Write theme statements (benefit + discriminator + proof)
5. Map to sections (where to emphasize each theme)
6. Create action captions (visual reinforcement)
7. Validate (quality checklist - benefit? differentiator? proof?)
